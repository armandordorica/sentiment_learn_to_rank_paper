5 · RavenPack Fine-Tuning

Ported from Streamlit app.py → render_ravenpack_finetuning_tab(), section 5·8 "Train / re-train the checkpoint". Select one ticker for a single-stock baseline, five for a multi-stock pilot, or more for a pooled run — all selected tickers are concatenated into one shuffled dataset, not trained sequentially.

Section 5 research summary: data contract, workflow, assumptions, and risks
Goal

Adapt a PhraseBank DistilBERT sentiment classifier to RavenPack financial headlines, select checkpoints without test-set tuning, and measure temporal and cross-stock generalization.

Primary inputs and schema

data/raw/news/ravenpack/{ticker}_articles_2003_2014.parquet. Fields consumed: ticker: string, headline: string, article_date: timestamp (or fallback article_time: timestamp), event_sentiment_score: float64, and story_id: string (or rp_story_id: string). Additional market and topic columns may be present but are not model features.

Initialization source

data/models/phrasebank_distilbert_best/: model.safetensors, config.json, tokenizer files, and metrics.json. The classifier consumes headline text only; RavenPack metadata supplies labels and split/audit keys.

Outputs and schema

data/models/ravenpack_distilbert_best/ contains weights and tokenizer artifacts plus metrics.json with tickers: list[string], dataset_paths: list[string], hyperparameters, validation/test: {eval_loss, eval_accuracy, eval_f1, ...}, per_ticker_test: map[ticker, metrics], device, timestamps, and W&B identifiers. provenance.json records reproducibility metadata.

What the notebook does

notebooks/finetune_on_ravenpack.ipynb calls the same shared pipeline as this page: discovers local exports, normalizes and deduplicates headlines, converts RavenPack scores to negative/neutral/positive labels, creates temporal splits, tokenizes text, fine-tunes DistilBERT, selects by validation macro-F1, evaluates once on test data, and saves artifacts. The web app adds background execution, pause/resume, caching, and live W&B links.

Core assumptions

RavenPack event sentiment is a valid three-class target; a headline is sufficient input; train ≤2011, validation=2012, and test ≥2013 represent deployment order; global story/headline deduplication prevents leakage; PhraseBank and RavenPack label IDs are aligned.

Risks

Vendor-label noise and threshold choice can bias results; repeated or syndicated stories may evade normalization; company/time coverage is uneven; temporal drift can lower future performance; pooled training can hide weak stocks; hyperparameter iteration against reported test scores would contaminate the holdout; cached benchmarks are valid only for unchanged hashes and source files.
